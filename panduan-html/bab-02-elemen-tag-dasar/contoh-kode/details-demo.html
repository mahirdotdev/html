1. Apa itu <details> dan <summary>?

Tag <details> membuat konten yang dapat dibuka/tutup (collapsible), sedangkan <summary> adalah judul yang dapat diklik untuk toggle konten.

Informasi tentang HTML

HTML (HyperText Markup Language) adalah bahasa markup yang digunakan untuk membuat struktur halaman web. HTML menggunakan tag-tag untuk menandai berbagai elemen seperti heading, paragraf, link, dan gambar.

6. Detail dengan Code Block

💻 Contoh Kode HTML Sederhana

Berikut adalah template dasar HTML yang bisa Anda gunakan:

<!DOCTYPE html>
<html lang="id">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Judul Halaman</title>
</head>
<body>
    <h1>Heading Utama</h1>
    <p>Paragraf pertama dengan <strong>teks tebal</strong>.</p>

    <h2>Sub Heading</h2>
    <ul>
        <li>Item pertama</li>
        <li>Item kedua</li>
        <li>Item ketiga</li>
    </ul>

    <p><a href="https://www.google.com">Link ke Google</a></p>
</body>
</html>

Cara menggunakan:

  1. Copy kode di atas
  2. Paste ke VS Code
  3. Save dengan ekstensi .html
  4. Buka di browser
🎯 Tips Debugging HTML

Masalah Umum dan Solusinya:

Tag tidak berfungsi
  • Periksa typo pada nama tag
  • Pastikan tag ditutup dengan benar
  • Check proper nesting
Karakter aneh muncul
  • Tambahkan <meta charset="UTF-8">
  • Save file dengan encoding UTF-8
  • Gunakan HTML entities untuk karakter khusus
Layout berantakan
  • Gunakan Developer Tools (F12)
  • Check element structure
  • Validate HTML di validator.w3.org

8. Accessibility dan Browser Support

♿ Accessibility Features

Tag details/summary memiliki built-in accessibility features:

  • Keyboard navigation: Bisa diakses dengan Tab dan Enter/Space
  • Screen reader friendly: ARIA attributes otomatis
  • Focus management: Browser menangani focus secara otomatis
  • Semantic meaning: Jelas bahwa ini adalah collapsible content

Untuk accessibility yang lebih baik:

<details role="group">
    <summary role="button" aria-expanded="false">
        Click untuk expand
    </summary>
    <div role="region">
        Konten yang dapat di-collapse
    </div>
</details>

Fallback untuk Browser Lama:

/* CSS untuk fallback */
details:not([open]) > *:not(summary) {
    display: none;
}

details > summary {
    cursor: pointer;
}

/* JavaScript fallback */
if (!('open' in document.createElement('details'))) {
    // Implement custom toggle functionality
}

9. Best Practices

✅ Do:

  • Gunakan summary yang descriptive dan jelas
  • Kelompokkan konten related dalam satu details
  • Berikan visual feedback (hover, focus states)
  • Test keyboard navigation
  • Pertimbangkan mobile experience

❌ Don't:

  • Jangan nest terlalu dalam (max 2-3 level)
  • Jangan buat summary yang terlalu panjang
  • Jangan gunakan hanya untuk visual styling
  • Jangan lupa test di berbagai browser

💡 Tips:

  • Gunakan CSS untuk styling yang konsisten
  • Pertimbangkan menggunakan JavaScript untuk advanced functionality
  • Test dengan screen readers
  • Provide fallback untuk browser yang tidak mendukung
