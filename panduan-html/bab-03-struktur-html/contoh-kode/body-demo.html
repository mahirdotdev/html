üí° Apa itu Semantic HTML?

Semantic HTML menggunakan tag yang memiliki makna, bukan hanya untuk styling. Ini membantu:

  • üîç SEO - Search engines memahami struktur konten
  • ‚ôø Accessibility - Screen readers dapat navigasi dengan mudah
  • üë®‚Äçüíª Maintainability - Code lebih mudah dipahami developer

Struktur Body Halaman Ini:

<body> <header role="banner"> <!-- Logo dan navigasi utama --> </header> <main role="main"> <!-- Konten utama halaman --> <section> <!-- Bagian-bagian konten --> </section> </main> <aside role="complementary"> <!-- Konten sampingan/sidebar --> </aside> <footer role="contentinfo"> <!-- Footer informasi --> </footer> </body>

‚ôø Fitur Accessibility

1. Skip Links

Coba tekan Tab di keyboard - akan muncul "Skip to main content" untuk user screen reader.

<a href="#main-content" class="skip-link">Skip to main content</a>

2. ARIA Roles dan Labels

Halaman ini menggunakan ARIA roles untuk memberikan konteks tambahan:

  • role="banner" - Header utama
  • role="main" - Konten utama
  • role="navigation" - Area navigasi
  • aria-label - Label deskriptif
  • aria-current="page" - Halaman aktif

3. Keyboard Navigation

Semua elemen interaktif dapat diakses dengan keyboard:

  • Tab - Navigasi maju
  • Shift + Tab - Navigasi mundur
  • Enter - Aktivasi link/button

‚úÖ Best Practices Tag Body

‚úÖ Do:

  • Gunakan semantic HTML elements
  • Sertakan skip links untuk accessibility
  • Struktur konten secara logical
  • Letakkan scripts di akhir body
  • Gunakan proper heading hierarchy
  • Test dengan keyboard navigation

‚ùå Don't:

  • Gunakan deprecated attributes (bgcolor, etc)
  • Letakkan semua JavaScript di head
  • Lupa struktur semantic
  • Skip heading levels (h1 ‚Üí h3)
  • Ignore accessibility guidelines
  • Use div for everything
